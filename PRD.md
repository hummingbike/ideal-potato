# PRD — 나혼자만 인테리어 (Grid Interior Planner)

> 상태: 초안 (Draft) · 최종 수정: 2026-06-17

## 1. 한 줄 정의
내 방의 평면도를 그리드로 나누고, 각 칸을 실제 사진으로 채운 뒤, 웹에서 가구를 자유롭게 옮겨보고 AI 추천 배치까지 받아보는 **개인용 인테리어 시뮬레이터**.

## 2. 배경 / 문제
- 가구 배치를 바꾸고 싶을 때마다 실제로 가구를 옮겨보기 전에는 결과를 가늠하기 어렵다.
- 줄자로 실측하고 평면도를 손으로 그리는 과정은 번거롭고, 기존 인테리어 앱은 가구를 직접 모델링/구매해야 해서 "내가 이미 가진 가구"를 그대로 반영하기 어렵다.
- 내 방을 사진으로 찍은 그대로 가상으로 재배치해볼 수 있는, 가볍고 개인적인 도구가 필요하다.

## 3. 타겟 사용자
- 1인 사용자 (본인) 기준으로 시작하는 개인 서비스. 멀티유저/공유 기능은 범위 밖.
- 자취/1인 가구 등 방 1~2개 규모의 공간을 대상으로 가정 (초기 범위).

## 4. 핵심 가치 제안
1. **내 공간 그대로**: 추상적인 3D 모델이 아니라 실제 내 방/가구 사진 기반.
2. **그리드로 단순화**: 정밀 3D 스캔 없이, 평면도를 격자로 나눠 칸 단위로 다루는 가벼운 접근.
3. **드래그 재배치 + AI 추천**: 직접 옮겨보거나, AI가 동선/공간활용 기준으로 배치를 제안.

## 5. 핵심 사용자 플로우
1. **평면도 입력**: 평면도 이미지를 업로드하거나, 직접 그리기 툴 + 치수 입력 폼(가로/세로 또는 벽별 길이)으로 정확한 치수의 외곽선을 생성한다.
2. **그리드 분할**: 가로/세로 칸 수(또는 칸 크기, 예: 0.5m 단위)를 지정해 평면도 위에 격자를 생성한다.
3. **칸 채우기**: 각 그리드 칸을 탭하면 카메라/사진 업로드로 그 위치의 실제 사진을 등록한다. (벽, 가구, 빈 공간 등)
4. **가구 식별**: 등록된 사진에서 자동 세그멘테이션으로 가구 영역을 인식·추출하고, 결과를 2D 탑뷰 아이콘으로 변환한다.
5. **재배치**: 웹 화면에서 가구 오브젝트(2D 탑뷰 아이콘)를 자체 구현한 범용 캔버스 라이브러리 기반 그리드 위에서 드래그 앤 드롭으로 옮긴다. 칸 단위 스냅, 충돌(겹침) 표시를 지원한다.
6. **AI 추천 배치**: 현재 가구 목록과 방 크기를 바탕으로 AI가 1개 이상의 추천 배치안을 제시하고, 한 번에 적용/되돌리기가 가능하다.
7. **저장/비교**: 배치안을 여러 개 저장해두고 전후 비교(스냅샷)를 볼 수 있다.

## 6. 기능 요구사항

### MVP (Phase 1)
- [ ] 평면도 업로드 또는 치수 입력 폼 기반 외곽선 생성
- [ ] 그리드 크기 설정 (행/열 수, 또는 셀 크기)
- [ ] 셀별 사진 업로드 (모바일 카메라 촬영 포함)
- [ ] 사진에서 자동 세그멘테이션으로 가구 인식/추출 → 2D 탑뷰 아이콘화
- [ ] 자체 구현한 범용 캔버스 라이브러리 기반 그리드 보드에서 가구 아이템 드래그 앤 드롭 재배치
- [ ] 칸 단위 스냅 + 겹침(충돌) 표시
- [ ] 배치 저장 / 불러오기 (최소 1개 레이아웃)

### Phase 2 — 가구 인식 고도화
- [ ] 자동 세그멘테이션 결과 확인/보정 UI (오인식 수정, 재크롭)
- [ ] 배경 제거 품질 개선 (투명 PNG 정제)
- [ ] 가구 메타데이터 입력 (이름, 크기(칸 수), 카테고리)

### Phase 3 — AI 추천 배치
- [ ] 현재 가구 목록 + 방 크기 + (선택) 사용자 선호 조건을 입력으로 추천 배치 생성
- [ ] 추천 사유 텍스트 설명 (예: "창가 동선 확보, 침대-책상 거리 최소화")
- [ ] 추천안 미리보기 → 적용 / 다른 추천 보기

### Phase 4 — 다듬기
- [ ] 여러 레이아웃 비교 뷰 (전/후 스냅샷)
- [ ] 레이아웃 공유용 이미지 내보내기 (개인 보관/SNS 공유 목적)
- [ ] PWA/모바일 대응 (촬영 → 즉시 칸 채우기 흐름 최적화)

## 7. 비기능 요구사항
- 모바일 브라우저에서 카메라 촬영 → 업로드 흐름이 매끄러워야 한다 (촬영 후 바로 셀에 매핑).
- 사진/가구 데이터는 개인 소유 공간이므로 비공개 저장이 기본.
- 그리드 보드 인터랙션(드래그, 스냅)은 입력 지연 없이 즉각 반응해야 한다.
- 초기에는 단일 사용자 기준이므로 동시성/권한 모델은 단순하게 유지.
- 그리드/드래그앤드롭 캔버스 라이브러리는 이 프로젝트 전용 코드에 의존하지 않고 다른 프로젝트에서도 재사용 가능한 범용 API로 설계한다.

## 8. 데이터 모델 (초안)
- **Floorplan**: id, 이름, 원본 이미지(또는 외곽선 좌표 + 치수 입력 데이터), 생성일
- **Grid**: floorplan_id, 행 수, 열 수, 셀 크기(실측 단위)
- **Cell**: grid_id, row, col, photo_url(nullable)
- **FurnitureItem**: id, 이름, 카테고리, 크기(가로 칸 수 x 세로 칸 수), 원본 이미지 url, 세그멘테이션 결과(마스크/크롭) url, 2D 아이콘 이미지 url, 소속 cell(최초 등록 위치)
- **Layout**: id, floorplan_id, 이름, 생성일, FurnitureItem별 (row, col) 배치 좌표 목록
- **AIRecommendation**: layout_id(추천 결과로 생성된 레이아웃), 추천 사유 텍스트, 생성 시각

## 9. AI 추천 로직 — 접근 방식 (미확정, 단계적 적용)
- **v1 (규칙 기반)**: 가구 크기/카테고리 기반 휴리스틱 (예: 큰 가구는 벽에 붙이기, 문/창 앞 칸은 비우기, 동선 폭 최소 N칸 확보).
- **v2 (LLM 활용)**: 방 크기, 가구 목록, 제약 조건을 텍스트/구조화 데이터로 Claude API에 전달 → 배치 좌표 + 추천 사유 생성.
- 자동 인식이 들어가는 시점에 따라 v1과 v2 도입 순서는 조정 가능.

## 10. 성공 기준 (개인 프로젝트 기준)
- 평면도 입력 → 첫 레이아웃 저장까지 소요 시간이 충분히 짧다 (체감 기준, 정량 KPI는 추후).
- AI 추천 배치가 최소 "말이 되는" 수준으로 동작 (벽/창문과 겹치지 않음, 가구 간 겹침 없음).
- 실제로 본인이 방 배치를 바꿀 때 한 번이라도 참고용으로 사용하게 되는 것.

## 11. 범위 외 (Out of Scope, 현재 단계)
- 다중 사용자/계정 공유, 인테리어 업체 연계, 가구 구매/커머스 연동
- 정밀 3D 렌더링 (그리드 기반 2D/탑뷰로 충분)
- 실시간 협업 편집

## 12. 결정된 사항 (Resolved Decisions)
- **가구 인식**: 처음부터 자동 세그멘테이션 사용. 수동 크롭은 보정(오인식 수정) 단계에서만 보조적으로 사용.
- **그리드 보드 표현**: 순수 2D 탑뷰 아이콘 사용 (실제 사진 썸네일을 그대로 가구로 쓰지 않음).
- **캔버스 라이브러리**: `react-konva`/`dnd-kit` 같은 외부 라이브러리 대신 직접 구현. 이 프로젝트 외 다른 곳에서도 쓸 수 있도록 범용 패키지로 설계 (자세한 내용은 [PLAN.md](PLAN.md) 참고).
- **평면도 외곽선 입력**: 손으로 그릴 때 정확한 치수 입력 폼을 함께 제공.
- **백엔드/스토리지**: Supabase (Postgres + Storage + Auth) 확정.
- **보정 UI 범위**: 자동 세그멘테이션 오인식 시, 박스 드래그로 인식 영역(위치/크기)만 재조정. 마스크 직접 편집(브러시 보정)은 지원하지 않음.

## 13. 남은 미해결 질문 (Open Questions)
- 자동 세그멘테이션 모델/서비스 선정 (자체 호스팅 vs 외부 API) — 지금은 보류, Phase 1~2 사이 PoC에서 캔버스 라이브러리/MVP 구현과 함께 짧게 검증 후 결정.
